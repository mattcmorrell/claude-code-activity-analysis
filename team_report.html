How does this work?

Prompt caching: Claude Code saves your conversation in a cache so it doesn’t have to re-read everything on every turn. By default, this cache expires after 5 minutes of inactivity. You can extend it to 1 hour by adding "CLAUDE_CODE_USE_EXTENDED_CACHE_TTL": "true" to your Claude Code settings.

Cache expiry: When the cache expires (you were idle too long), Claude has to rebuild it from scratch. This is expensive — writing to cache costs 12.5× more than reading from it. This is the main source of “wasted” spend.

Why big sessions cost more: Every turn reads your full conversation. A session at 200K tokens reads all 200K per turn. At $0.50/MTok, that’s $0.10 just in reads — before Claude even responds.

The 5-minute rule: If you’re in a long session and need a break, either extend the cache timeout to 1 hour (see above), or type /compact before stepping away to shrink the session. If you’ll be gone a while, /clear starts fresh — cheaper than Claude reloading a huge conversation.
